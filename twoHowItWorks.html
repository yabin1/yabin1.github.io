How it works...

This frameworks is based on a MVC pattern, and help you to map your view calls into a xml file config for both directions. You need to create your badger-config.xml, this is the first step to map your html views and the java methods code without the needy to use a Server between this technologies

Creating your badger-config.xml

This file is created to set a behavior into the web views. Please check the basic config for this.
First you need to create into your project->assets, a new xml filewith the name: badger-config.xml

badger-config.xml

The basic information for badger-config.xml is this:


			
		  

<webviews>
  <view-services>
    <view-service>
	  <id-WebView>homeView</id-WebView>
	  <class-name>com.honey.jsService.Gen</class-name>
	  <html-interface-name>MyView</html-interface-name>
	  <url>file:///android_asset/web/home.html</url>  
	  <map></map>
	 </view-service>
  </view-services>
</webviews>
